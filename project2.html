writeAM Text Editor


Overview and Goals :

The goal of this project was to build a lightweight, minimalist text editor that runs entirely in the terminal. Named "writeAM" after my initials, this project was an exercise in systems programming, built from scratch in C with no external library dependencies to deepen my understanding of memory management, terminal I/O, and application architecture.


Inspiration:

This project was heavily inspired by the classic "Build Your Own Text Editor" tutorial for the Kilo editor. I was motivated to create a tool that embodies the simplicity and power of traditional Unix command-line utilities. The design focuses on a clean, functional, terminal-based UI, using raw mode and ANSI escape sequences to control rendering and user interaction directly.


Challenges :

Building an editor in pure C presented several challenges. A primary one was managing the terminal in raw mode, which required careful handling of low-level I/O to process keypresses individually instead of line by line. Implementing the syntax highlighting engine, which parses code to color keywords, comments, and strings, was another complex task that demanded meticulous string manipulation and state management. Finally, ensuring robust memory management for file buffers and text rows was critical to prevent leaks and crashes.


Key Features :

  • Minimalist Terminal UI: A clean, distraction-free writing environment that runs directly in the terminal.
  • Syntax Highlighting: Provides syntax highlighting for the C programming language, including keywords, comments, strings, and numbers.
  • Text Search: Features an incremental search function (Ctrl+F) to quickly find text within a file.
  • Zero Dependencies: Built with pure C and standard libraries, making it highly portable across Unix-like systems.


Learning Outcomes :

This project was an invaluable learning experience in C programming and low-level systems interaction. I gained a practical understanding of terminal I/O, including manipulating raw mode and using escape sequences. It significantly improved my skills in manual memory management, pointer arithmetic, and building robust data structures. Developing features like search and syntax highlighting also provided deep insights into text processing algorithms and state machine design.
