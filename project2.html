writeAM Text Editor


Overview & Goals

writeAM is a lightweight, zero-dependency text editor running entirely in the terminal. Named after my initials, this project was a deep dive into systems programming in C.

The goal was to demystify how command-line tools function under the hood by building a functional editor from scratch, bypassing high-level abstractions to interact directly with system memory and terminal input/output streams.


Inspiration

Inspired by the legendary reliability of tools like Vim and Nano, and guided by the logic of the Kilo editor, I wanted to create a tool that embodies the Unix philosophy: do one thing and do it well. The design focuses on raw efficiency, using ANSI escape sequences to handle rendering without the bloat of graphical libraries like ncurses.


Technical Challenges

Terminal I/O & Raw Mode: The most significant hurdle was manipulating the terminal's termios struct to switch from canonical mode to raw mode. This allowed the program to process input byte-by-byte (essential for mapping custom keybindings) rather than waiting for a newline delimiter.

Dynamic Memory Management: Handling file buffers required rigorous manual memory management. I implemented dynamic arrays (reallocating memory as rows grow) to prevent buffer overflows while strictly managing `malloc` and `free` cycles to ensure zero memory leaks.


Key Features

  • Zero External Dependencies: Built strictly with the C Standard Library, ensuring high portability across Unix/Linux systems.
  • Syntax Highlighting Engine: A custom parser that detects and color-codes C-language keywords, strings, and comments in real-time.
  • Incremental Search: Implemented a search algorithm (Ctrl+F) that scans the file buffer and scrolls the viewport to matches dynamically.
  • Distraction-Free UI: A raw-mode interface featuring line numbering, status bars, and dirty-flag indicators for unsaved changes.


Engineering Outcomes

This project was a masterclass in C programming and pointer arithmetic. It sharpened my ability to debug segmentation faults and understand the stack vs. heap. Beyond the code, I gained a fundamental understanding of how software interacts with the OS kernel via standard input/output (STDIN/STDOUT), significantly improving my confidence in low-level systems architecture.
